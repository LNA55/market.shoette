The Skills used in Step 1
Read the Market

What Step 1 does, step by step — from raw input to the published market report. This page documents the version of the skill in force on 12 June 2026; its exact definition is downloadable at the bottom of the page.

What a run produces

One execution of the skill = one run: a frozen, dated report, published at market.shoette.com/[market]/s1-[N]_[date]/, together with its data.json — the structured-data canvas that feeds the interactive chart and will serve the following skills. With each run, the market's page and the home page are updated.

The report contains four blocks:

  • Executive Summary — the market framing in half a page: the dimensions that structure it (the reading grid for everything else), and the assumptions made on the input.
  • 2a — Market share & growth — the share × growth scatter plot, preceded by the maturity stage and the justified choice of reference period.
  • 2b — Interactive positioning — the 5-visual-channel chart (layers, player cards, annotated export), followed by the table of all the values and the “How to read these values” block (units, currencies, conversions).
  • Appendices — A1: all the sources (URL + access date); A2: the sector's lexicon, defined in French and English.

The process, step by step

Step 0 — Resolve the run's context

The skill identifies the market (ASCII slug, reused if the market already exists), computes the run number (latest S1 run + 1), sets today's date — hence the folder s1-[N]_[date] — and adopts the input's language for the whole report (the site's parent pages stay in French). A preliminary check verifies the interactive chart engine exists: without it, the skill refuses to start.

Step 1 — Interpret the input

The input can be a sector, a need expressed in natural language, a geographic area — or any combination. If it's incomplete, the skill infers the most likely market and displays its assumptions at the top of the report rather than asking. It only asks a clarifying question if the input is too ambiguous to proceed.

Step 2 — Research

Research subagents fan out in parallel across the web (international business press and the sector's specialized press) — they run on an economical model (Sonnet), with consolidation kept on the best model of the session. They establish: the market's maturity stage, which determines the reference period (6 months, 1 year or 3 years, a justified choice); the dimensions that structure the market, ranked by importance; and for each significant player, the estimated market share, growth and a value per dimension. Three absolute rules: every estimate is flagged, no significant player is silently omitted (lacking quantified data, it is listed separately with what was searched for), and raw data is not kept — only the structured data and the source references survive.

Step 3 — Build the data canvas

Everything is consolidated into a data.json with a fixed schema: the ranking of the dimensions drives the interactive chart's default configuration (rank 2 → X position + color, rank 3 → size, etc.); each value carries its “estimated” flag; the player and dimension identifiers are stable slugs, designed to be matched from one run to the next by the following skills; and the export note (reading key + first market observation) is written there to be embedded in the chart's PNG/SVG exports.

Step 4 — Generate the run page

The page is self-contained and frozen: the data is inlined and written into the neighboring data.json — strictly identical. The 2a chart is a static SVG generated for this run; the 2b is mounted on the site's shared, versioned engine (built once, never rebuilt during a run); below the chart, the table of all the values then the reading notes — unit and currency used, any conversions with exchange rate, meaning of the scales. The page ends with the appendices and the “last updated” field, the only element a future skill is allowed to modify.

Step 5 — Update the parent pages

The market page (Step 1 section) and the home page are updated only between their RUNS-SKILL1 and MARKETS tags — runs from oldest to newest, Run 1 first. Everything else on these pages belongs to Elena and is never overwritten. On a market's first run, the market page is created from the site template.

Step 6 — Deploy, verify, commit

Deployment in two stages: a simulation (--dry-run) to check the file list, then the real upload — to the hosting's market/ directory exclusively, a script guardrail rejecting any other path. The skill then verifies the pages respond online, then commits the run to Git: each run is versioned, the history is replayable.

The guardrails

  • Estimates always flagged — the “~” prefix follows every estimated value, from data.json to the exports.
  • Never a silent omission — a player with no quantified data stays visible, with a trace of what was searched for.
  • Frozen runs — a published run is never modified (apart from the “last updated” field, reserved for a future skill).
  • Strict remote scope — the skill writes only to the hosting's market/ directory.
  • Split models — research on Sonnet, orchestration and writing on the best model available.
  • Traceability — dated sources in the appendix, structured data versioned by Git, raw data discarded.
