Documentation · Step 3

Focus on Step 3
Position MY product in the Market

Le marché a été lu (Step 1) puis analysé (Step 2). La Step 3 inverse le regard : elle place un projet à moi au milieu des acteurs de ce marché, sur le graphique de positionnement déjà construit.

À quoi sert cette étape

Les deux premières étapes regardent le monde : qui sont les acteurs, combien ils pèsent, comment le marché se structure. La Step 3 y introduit un nouveau projet — le mien : elle l'évalue sur chacune des dimensions qui structurent le marché, et le pose sur le graphique interactif au milieu de ses futurs concurrents. On voit d'un coup d'œil de qui le projet est proche, de qui il s'éloigne, et quels espaces du marché restent vides.

Comme les autres étapes, chaque exécution produit un run daté et figé : une note de positionnement, listée sur la page du marché. Si le projet pivote ou si le marché bouge, on relance — et l'historique des positionnements reste consultable.

Comment la lancer

Dans Claude Code, entrer « lance la skill 3 » ou « positionne mon projet », avec les inputs suivants :

  • Le projet — une description libre, ou un lien vers sa source (par exemple le site du projet : l'agent le lit pour comprendre l'offre).
  • Le marché — l'agent propose la liste des marchés déjà étudiés ; on choisit d'un clic. On peut aussi coller directement les liens des runs Step 1 et Step 2 concernés.
  • En option — toute information complémentaire que j'ai sur le marché ; elle est citée comme telle dans la note.

Prérequis : le lancement de la Step 3 dépend de l'existence d'un run Step 2 sur le même sujet (en plus du run Step 1, qui porte le graphique). Si le marché n'a pas encore été passé par la Step 2, la skill s'arrête et propose de la lancer d'abord. Si la matière fournie ne suffit pas pour évaluer le projet sur les dimensions du marché, l'agent pose au plus 2-3 questions ciblées.

Ce que fait la skill

1 · Le projet entre dans le graphique de la Step 1

La skill évalue le projet sur chaque dimension du graphique de positionnement du dernier run Step 1 — mêmes échelles, mêmes conventions que les concurrents — puis l'ajoute au graphique interactif et au tableau de données qui l'accompagne. Le projet y porte un marqueur distinctif ✺ (rayons autour du cercle, nom en gras) : impossible de le confondre avec un acteur du marché.

L'enrichissement est strictement additif : les données des concurrents ne sont jamais retouchées, et la date d'origine du graphique reste intacte — une mention datée « Graphique enrichi des infos sur le nouveau projet [nom], le [date] » s'ajoute simplement à côté.

Sa part de marché vaut 0 % par définition — c'est un nouveau projet. La lecture intéressante se fait donc sur les autres dimensions : prix, modèle, ancrage, personnalisation… celles que le lecteur peut affecter aux canaux visuels du graphique.

2 · Une page dédiée à ce run

La skill génère aussi une note de positionnement — la page du run Step 3 — qui contient, dans l'ordre :

  • Le projet — son nom, sa description en quelques lignes, le lien vers sa source.
  • Le positionnement — un tableau compact : chaque dimension du marché, la valeur attribuée au projet, et la justification en une ligne ; puis le lien vers le graphique enrichi, dans le run Step 1 d'origine.
  • L'analyse du graphique — un paragraphe avec les informations importantes et les clés de lecture : où le projet atterrit, ses voisins immédiats, les espaces vides — et la configuration du graphique recommandée pour bien le voir (le projet étant à 0 % sur l'axe vertical par défaut).

Les valeurs du projet viennent de sa source ou de moi — elles sont déclaratives, et la note le dit explicitement, à la différence des valeurs des concurrents qui sont sourcées publiquement.

Où retrouver le run

Le lien vers la page du run apparaît dans la section Step 3 de la page principale du marché (par exemple Applications mobiles d'assistance à la perte de poids), comme les runs des autres étapes. Le graphique enrichi, lui, vit toujours à sa place d'origine : dans le run Step 1 du marché.

Les conventions de lecture

  • ✺ à côté d'un nom : mon projet — sur le graphique, dans l'infobulle et sur la fiche détaillée.
  • Part de marché du projet : 0 %, toujours — un nouveau projet n'en a pas encore ; sa croissance est notée « — » (non applicable).
  • Valeurs du projet déclaratives (sa source, ou moi) ; valeurs des concurrents sourcées ; ~ = estimation, comme partout sur le site.
  • Les runs restent figés : la Step 3 ne fait qu'ajouter son projet au graphique — rien d'existant n'est modifié, et la mention d'enrichissement est datée.
